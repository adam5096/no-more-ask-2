## 1. 語義化 HTML 決策

- [ ] **應該使用哪些語義化標籤？**
  - [ ] `<header>`：頁首、組件標題區
  - [ ] `<nav>`：導航區塊
  - [ ] `<main>`：主要內容區
  - [ ] `<section>`：內容區塊
  - [ ] `<article>`：獨立內容（如貼文、卡片）
  - [ ] `<aside>`：側邊欄、補充資訊
  - [ ] `<footer>`：頁尾

- [ ] **是否需要 `<main>` 標籤標示主要內容區？**
  - 頁面級組件應該使用 `<main>`
  - 子組件通常不需要

- [ ] **是否需要 ARIA 標籤？**
  - [ ] `aria-label`：為無文字的元素提供標籤
  - [ ] `aria-labelledby`：引用其他元素的標籤
  - [ ] `role`：明確指定元素角色

- [ ] **是否需要無障礙功能？**
  - [ ] `alt` 文字：所有圖片都需要
  - [ ] 鍵盤導航：可互動元素應該可以用鍵盤操作
  - [ ] 焦點管理：確保焦點順序合理

---

## 2. HTML 結構層級

- [ ] **結構層級是否過深（建議不超過 5 層）？**
  - 如果層級過深，考慮提取為子組件

- [ ] **是否需要提取為子組件？**
  - 判斷標準：是否有重複的 HTML 結構、是否有獨立的邏輯區塊

- [ ] **是否有重複的 HTML 結構可以抽象化？**
  - 如果有，考慮提取為組件或使用 `v-for`

---

## 3. 內容組織

- [ ] **內容是否按邏輯區塊分組？**
  - 使用適當的容器元素（`<div>`、`<section>`）分組

- [ ] **是否需要條件渲染？**
  - [ ] `v-if`：條件為 false 時不渲染（適合切換顯示/隱藏）
  - [ ] `v-show`：條件為 false 時隱藏但保留 DOM（適合頻繁切換）

- [ ] **列表渲染是否使用 `v-for` 與 `key`？**
  - ✅ 正確：`<div v-for="item in items" :key="item.id">`
  - ❌ 錯誤：缺少 `key` 或使用 `index` 作為 key（除非列表不會變動）
