## [x] 【前端元素】標題和關閉按鈕應正確顯示
**範例輸入**：組件掛載
**期待輸出**：顯示 "User Registration" 標題和 "X" 關閉按鈕

---

## [x] 【前端元素】表單欄位應正確渲染
**範例輸入**：組件掛載
**期待輸出**：顯示 email、password、firstName、lastName、displayName 五個輸入欄位及其對應的 label

---

## [x] 【前端元素】提交按鈕在提交中應顯示為 disabled 狀態
**範例輸入**：表單提交中（isSubmitting = true）
**期待輸出**：提交按鈕顯示 "註冊中..." 且為 disabled 狀態

---

## [x] 【前端元素】成功訊息應在註冊成功後顯示
**範例輸入**：API 返回成功回應
**期待輸出**：顯示 "註冊成功！歡迎加入" 訊息和 "確定" 按鈕，表單隱藏

---

## [x] 【前端元素】錯誤訊息應在欄位驗證失敗時顯示
**範例輸入**：email 欄位驗證失敗
**期待輸出**：在 email 輸入框下方顯示對應的錯誤訊息

---

## [x] 【function 邏輯】Email 欄位為空時應顯示必填錯誤
**範例輸入**：email 欄位為空，觸發 blur 事件
**期待輸出**：顯示 "Email 為必填欄位" 錯誤訊息

---

## [x] 【function 邏輯】Email 格式無效時應顯示格式錯誤
**範例輸入**：email 欄位輸入 "invalid-email"，觸發 blur 事件
**期待輸出**：顯示 "請輸入有效的 Email 格式" 錯誤訊息

---

## [x] 【function 邏輯】Password 欄位為空時應顯示必填錯誤
**範例輸入**：password 欄位為空，觸發 blur 事件
**期待輸出**：顯示 "密碼為必填欄位" 錯誤訊息

---

## [x] 【function 邏輯】Password 長度不足時應顯示長度錯誤
**範例輸入**：password 欄位輸入 "12345"（少於 6 個字元），觸發 blur 事件
**期待輸出**：顯示 "密碼長度至少為 6 個字元" 錯誤訊息

---

## [x] 【function 邏輯】Password 缺少特殊字元、大寫或小寫時應顯示錯誤
**範例輸入**：password 欄位輸入 "password123"（缺少特殊字元和大寫），觸發 blur 事件
**期待輸出**：顯示 "密碼須包含一個以上如下字元：特殊字元、英文字母大、小寫" 錯誤訊息

---

## [x] 【function 邏輯】First Name 欄位為空時應顯示必填錯誤
**範例輸入**：firstName 欄位為空，觸發 blur 事件
**期待輸出**：顯示 "First Name 為必填欄位" 錯誤訊息

---

## [x] 【function 邏輯】Last Name 欄位為空時應顯示必填錯誤
**範例輸入**：lastName 欄位為空，觸發 blur 事件
**期待輸出**：顯示 "Last Name 為必填欄位" 錯誤訊息

---

## [x] 【function 邏輯】Display Name 欄位為選填，不應顯示驗證錯誤
**範例輸入**：displayName 欄位為空，觸發 blur 事件
**期待輸出**：不顯示任何錯誤訊息

---

## [x] 【function 邏輯】表單提交時應驗證所有必填欄位
**範例輸入**：表單有多個必填欄位為空，點擊提交按鈕
**期待輸出**：所有空欄位都顯示對應的必填錯誤訊息，且不發送 API 請求

---

## [x] 【Mock API】API 成功回應時應顯示成功訊息
**範例輸入**：填寫完整且有效的表單資料，點擊提交按鈕，Mock API 返回成功回應
**期待輸出**：顯示成功訊息 "註冊成功！歡迎加入"，表單隱藏

---

## [x] 【Mock API】API 返回欄位級別錯誤時應顯示對應錯誤訊息
**範例輸入**：填寫完整且有效的表單資料，點擊提交按鈕，Mock API 返回欄位級別錯誤（如 email 已存在）
**期待輸出**：在對應欄位下方顯示 API 返回的錯誤訊息

---

## [x] 【Mock API】API 返回一般錯誤時應正確處理
**範例輸入**：填寫完整且有效的表單資料，點擊提交按鈕，Mock API 返回一般錯誤（非欄位級別）
**期待輸出**：isSubmitting 狀態解除，錯誤記錄到 console（不顯示 UI 錯誤訊息）

---

## [x] 【前端元素】關閉按鈕點擊時應觸發 close 事件
**範例輸入**：點擊關閉按鈕（X）
**期待輸出**：觸發 close emit 事件

---

## [x] 【前端元素】成功訊息中的確定按鈕點擊時應觸發 close 事件
**範例輸入**：註冊成功後，點擊成功訊息中的 "確定" 按鈕
**期待輸出**：觸發 close emit 事件
