# Auth Refactor 測試案例

> 狀態：初始為 [ ]、完成為 [x]
> 注意：狀態只能在測試通過後由流程更新。

---

## [x] 【邏輯層】usePasswordStrength 密碼檢查邏輯
**範例輸入**：`Password123!`
**期待輸出**：`length`, `special`, `upper`, `lower` 皆為 `true`

---

## [x] 【前端元素】AuthInput 渲染與狀態
**範例輸入**：傳入 `label="Email"`, `errors=["Email 為必填"]`
**期待輸出**：正確顯示 Label 標籤與錯誤響應樣式（紅框與錯誤文字）

---

## [x] 【前端元素】LoginForm 密碼提醒功能
**範例輸入**：進入登入頁面，展開密碼輸入框
**期待輸出**：應顯示密碼組成規則提醒（長度、大寫、特殊字元等），且為唯讀/裝飾狀態

---

## [x] 【驗證邏輯】RegisterForm 即時驗證同步
**範例輸入**：在註冊表單輸入密碼時
**期待輸出**：密碼強度標籤雲應根據輸入即時更新 `is-valid` 樣式

---

## [x] 【Mock API】表單提交與錯誤回饋
**範例輸入**：正確填寫後點擊提交
**期待輸出**：UI 顯示 Loading 狀態，API 回傳成功後觸發 `success`流程；若 API 回傳錯誤，AuthInput 應顯示對應欄位錯誤。
