# B1 救援功能 v1 驗收標準

**版本**: v1.0  
**日期**: 2026-02-02  
**依據**: ORCA Discovery 訪談結果

---

## 功能驗收標準

### 建立救援請求

- [ ] 用戶可以進入「建立救援請求」頁面
- [ ] 表單包含所有必填欄位：預約日期、時長、壓力等級、地點、需要的幫助
- [ ] 表單包含選填欄位：場景描述、惱人問題列表、備註
- [ ] 壓力等級以 1-5 視覺化呈現 (1 輕微, 5 煎熬)
- [ ] 惱人問題可動態新增/刪除
- [ ] 必填欄位驗證正確運作
- [ ] 表單送出後建立請求成功

### 請求詳情頁

- [ ] 正確顯示請求所有資訊
- [ ] 顯示請求狀態 (draft/pending/confirmed/completed/cancelled)
- [ ] 顯示已選擇的 Helper 資訊 (如有)
- [ ] pending 狀態可取消請求
- [ ] 取消請求顯示確認對話框

### 請求列表 (Dashboard)

- [ ] Dashboard 正確顯示用戶的救援請求
- [ ] 點擊請求可導航到詳情頁
- [ ] 顯示請求狀態標籤

### Helper 選擇 (手動配對)

- [ ] 用戶可以瀏覽可用 Helper 列表
- [ ] Helper 卡片顯示：暱稱、評分、頭像
- [ ] 用戶可以選擇一個 Helper
- [ ] 選擇後請求狀態更新為 pending

- [ ] 必填欄位有視覺標記

- [ ] 手機版表單可正常操作
- [ ] 桌面版利用更寬的空間

## 技術驗收標準

### BFF API

- [ ] `POST /api/rescue` 建立救援請求
- [ ] `GET /api/rescue/:id` 取得請求詳情
- [ ] `PATCH /api/rescue/:id` 更新請求
- [ ] `PATCH /api/rescue/:id/status` 更新狀態
- [ ] `DELETE /api/rescue/:id` 取消請求 (軟刪除)

### 資料結構

- [ ] 所有 API 符合 `types/rescue.ts` 定義
- [ ] 日期使用 ISO 8601 格式

### 前端架構

- [ ] 使用 composable 封裝救援相關邏輯
- [ ] 表單驗證使用統一機制
- [ ] 錯誤處理統一格式
