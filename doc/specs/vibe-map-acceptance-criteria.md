# 情緒地景 (Vibe Map) v1 驗收標準

**版本**: v1.0  
**日期**: 2026-02-03  
**依據**: vibe-map-orca-discovery.md

---

## 功能驗收標準

### 發布情緒標籤

- [ ] 用戶可以在地圖上點擊位置發布心情標籤
- [ ] 表單包含：內容、分類 (焦慮/吐槽/炫耀/困擾)、壽命、匿名開關
- [ ] 壽命可選範圍：30 ~ 720 分鐘，預設 120 分鐘
- [ ] 發布後標籤立即出現在地圖上
- [ ] 匿名模式下不顯示暱稱

### 標籤顯示與互動

- [ ] 地圖正確顯示所有有效標籤
- [ ] 標籤根據分類顯示不同顏色/樣式
- [ ] 點擊標籤可查看詳情 (內容、發布者、剩餘時間)
- [ ] hover 時標籤有視覺回饋

### 生命週期 (蒸發機制)

- [ ] 標籤隨時間自動蒸發
- [ ] 剩餘 25% 壽命時開始縮小
- [ ] 剩餘 10% 壽命時明顯縮小
- [ ] 壽命結束後標籤消失

### 個人演化 (Evolution)

- [ ] 同一用戶同類型標籤累積 >= 10 個觸發演化
- [ ] 演化體以特殊樣式顯示 (焦慮雲 / 金光圈)
- [ ] 演化體位置為該用戶標籤的中心點

### 圖層控制

- [ ] 可切換「所有標籤」/「我的標籤」/「僅吐槽」圖層
- [ ] 圖層切換即時生效
- [ ] 圖層狀態持久化

---

## 設計驗收標準

### 視覺呈現

- [ ] 標籤以「氣泡」或「標籤雲」樣式呈現
- [ ] 不同分類有明確視覺區分
- [ ] 演化體樣式醒目且有動態效果
- [ ] 地圖整體視覺乾淨不擁擠

### 動畫效果

- [ ] 標籤出現時有「浮現」動畫
- [ ] 標籤縮小過程平滑過渡
- [ ] 演化觸發時有特效提示

### 響應式

- [ ] 手機版可正常操作地圖
- [ ] 發布表單在手機上可用

---

## 技術驗收標準

### BFF API

- [ ] `GET /api/vibe/stream` 回傳有效標籤與演化體
- [ ] `POST /api/vibe/create` 建立新標籤
- [ ] API 回傳包含 `remainingLifePercent` 供前端計算縮放

### 資料結構

- [ ] 所有 API 符合 `types/map.ts` 定義
- [ ] 時間戳使用 ISO 8601

### 前端架構

- [ ] 使用 Leaflet 自訂標記圖層
- [ ] 使用 composable 封裝 vibe 邏輯
- [ ] 標籤狀態每 30 秒自動更新
